# PRD — P-20260624-002: V40-V45 Release Operations CLI and API

## 1. Background
V37-V39 built a pure release operations center for failed gates, commit manifests, and remediation Markdown. Operators now need this surface available from CLI and Web API so cron, CI, and agent workflows can consume it without opening the browser.

## 2. Features

### V40 CLI release-ops summary
- Add `rdma release-ops` command.
- Print failed gate queue, commit manifest counts, and remediation Markdown.

### V41 CLI JSON mode
- Add `rdma release-ops --json` for machine-readable automation.
- Include `failedGateQueue`, `commitManifests`, and `remediationMarkdown`.

### V42 Proposal filter
- Add `--proposal <id>` to filter histories and manifests to one proposal.
- Keep output deterministic when the proposal has no release history.

### V43 Web API release-ops
- Add `GET /api/release-ops` to return the same model as CLI JSON.
- Use existing local proposal and release-history readers.

### V44 Web API proposal filter
- Add `GET /api/release-ops?proposal=<id>`.
- Validate by middleware unit test.

### V45 Fix prompt handoff
- Add copy-ready fix prompt text based on failed gates.
- Include gate label, history path, checklist, and verification command hints.
