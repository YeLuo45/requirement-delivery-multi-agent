# PRD — P-20260624-004: V52-V57 Delivery Closure Automation

## 1. Background
The previous release-ops iteration added PR draft and staging suggestions. The next gap is a fully machine-consumable delivery closure loop: automation needs stable JSON, CI summary markdown, and safe proposal status suggestions without mutating MCP/proposal state.

## 2. Features

### V52 Stable JSON contract
- Add `schemaVersion` to release-ops payload.
- Include `stageCommands`, `prDraftMarkdown`, and status suggestions in JSON output.

### V53 CI summary markdown
- Add `rdma release-ops --ci-summary`.
- Render GitHub Actions step-summary friendly markdown.

### V54 Safe status suggestions
- Compute safe next status actions from current proposal status and failed gates.
- Suggest `test_failed` when gates fail, `accepted` when in acceptance and clean, `deployed`/`delivered` for later states.
- Do not mutate proposal status.

### V55 CLI flags
- Add `--ci-summary` and ensure `--json` includes all automation fields.

### V56 README sync
- Document JSON/CI/status suggestion usage in README and README.zh-CN.

### V57 TDD and hard gates
- Cover helpers through public release-ops API tests.
- Keep zero new dependencies.
