# PRD — P-20260624-001: V37-V39 Release Operations Center

## 1. Background
V31-V36 made local release history readable from Web and API surfaces. Operators still need a faster acceptance center: failed gates should be queued first, commit preparation should be visible without opening JSON, and fix notes should be copy-ready for handoff.

## 2. Features

### V37 Failed gate queue
- Build a pure release operations model that extracts failed gates across persisted release histories.
- Sort by newest generatedAt first, then proposal id for deterministic output.
- Include proposal title, gate label, history path, and checklist.

### V38 Commit manifest summary
- Summarize latest release ownership groups into source/test/docs/generated/other counts.
- Expose recommended staging paths from the latest release history.
- Keep generated README demo JSON separate from ordinary dirty files.

### V39 Copy-ready remediation Markdown
- Render a Markdown handoff containing failed gates and checklist bullets.
- Return a clear empty-state Markdown when no failures exist.
- Surface release operations on the Web Release History page.
