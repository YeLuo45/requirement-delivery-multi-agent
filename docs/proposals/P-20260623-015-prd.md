# PRD — P-20260623-015: V15-V21 Delivery Audit Automation Batch

## 1. Background
V8-V14 added the shared acceptance evidence API, report helpers, side-effect classifier, and `release:local`. The next batch should make this audit surface easier to consume from Web, automation, and commit preparation.

## 2. Features

### V15 Web delivery report page
- Add `/delivery-report/:id` route and nav entry.
- Render a report-oriented page with gate summary, safe next actions, and changed-file groups.

### V16 JSON output for release:local
- Support `npm run release:local -- --json`.
- Emit a stable JSON payload with planned gates and dirty-file summary.

### V17 README side-effect isolation
- Make `verify:readme` use an isolated storage root for README demo commands when not explicitly provided.
- Avoid writing demo artifacts into the repository root where possible.

### V18 Web safe proposal state buttons
- Surface safe next actions on proposal detail-compatible data/page.
- Actions are display-only in this batch; no mutation endpoint.

### V19 Local acceptance history ledger
- Add deterministic release history record builder for `release:local` outputs.
- Keep history under `artifacts/release-local/` when command is run for real.

### V20 Failed gate deterministic fix checklist
- Convert gate drilldown into deterministic fix checklist text.
- No LLM dependency.

### V21 Diff ownership report
- Group dirty files by proposal/document/source/test/generated ownership.
- Make this available to `release:local` JSON payload.
