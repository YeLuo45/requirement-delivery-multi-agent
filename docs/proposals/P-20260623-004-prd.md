# PRD — P-20260623-004: V6 Web mode reaches TUI feature parity

## Background
The RDMA TUI exposes five operator capabilities: list snapshot, proposal detail, per-agent config, local proposal creation, and control-plane summary. Web mode already has list/detail/create/control-plane fragments, but these are not presented as a complete TUI-equivalent operator surface.

## Scope

### F-1 Web TUI parity model
- Add a pure Web parity summary that enumerates every TUI capability.
- Capabilities: list, show, config, new, control-plane.
- Each capability includes TUI command, Web route/API, and availability status.

### F-2 Web operator console
- Add a discoverable `/operator` route in the main navigation.
- The operator page shows TUI-equivalent capabilities, storage summary, recent proposals, and action links.
- It must not hide core features inside secondary flows.

### F-3 Config parity
- Add a Web-facing config panel/page backed by `/api/config`.
- Show the same per-agent mode/source/prompt state as `rdma tui --config`.

### F-4 README deliverability
- Document Web parity commands/routes in README.
- Keep README command verifier green with `Failed: 0`.
