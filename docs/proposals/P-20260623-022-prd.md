# PRD — P-20260623-022: V25-V30 Release History and Commit Manifest Automation

## 1. Background
V22-V24 made `release:local --json` parameterized and optionally persistent. The next batch should make persisted release evidence useful to operators, CI, and commit preparation.

## 2. Features

### V25 Release history reader
- Read `artifacts/release-local/*.json` from a configurable root.
- Sort records newest first and expose a compact summary model.

### V26 Actual gate result model
- Add deterministic result helpers for gate status, duration, exit code, and fix checklist.
- Keep shell execution behavior unchanged in this batch.

### V27 Release diff compare
- Compare two release history payloads by dirty files, gates, proposal metadata, and ownership.
- Return a machine-readable diff model.

### V28 Commit preparation manifest
- Convert ownership groups into explicit recommended staging paths.
- Separate source/test/docs/generated/other groups.

### V29 CI artifact mode
- Add `--ci` JSON mode metadata and summary markdown for CI artifacts.

### V30 Web delivery report history integration
- Add pure model that combines proposal evidence with latest release history for a proposal.
- Keep Web page dependency-light and testable through model helpers.
