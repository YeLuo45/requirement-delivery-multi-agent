# PRD — P-20260623-009: V7 web acceptance evidence dashboard

## 1. Background
V6 made Web mode cover TUI feature parity, but delivery acceptance evidence still lives mostly in CLI output, README verification, and terminal reports. Boss-facing validation should be visible on the Web overview/home surface.

## 2. Features

### F-1 Acceptance evidence model
- Add a pure Web-side model that summarizes acceptance gates from proposal-like records.
- Gate types: tests, coverage, README verification, build.
- Output counts: total gates, passed gates, failed gates, pass rate, latest evidence timestamp.

### F-2 Overview dashboard panel
- Show acceptance evidence on the main overview page, not hidden in a subflow.
- Include recent accepted/deployed/delivered proposals with gate summary.
- Keep empty states actionable.

### F-3 API/demo compatibility
- Work with existing proposal JSON shapes and demo fallback data.
- Avoid new dependencies.
