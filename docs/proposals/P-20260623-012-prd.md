# PRD — P-20260623-012: V8-V14 Delivery Automation Completion Batch

## 1. Background
V7 made acceptance evidence visible on the Web overview. Boss now requested completion of all follow-up directions in one batch: API unification, report export, Web state actions, failure drilldown, README side-effect cleanup, proposal/code association, and a local release gate.

## 2. Features

### V8 Acceptance evidence API
- Add `/api/acceptance-evidence` backed by the same `acceptance-evidence` model used by Web Overview.
- Return summary + rows in JSON.

### V9 One-click delivery report export
- Add pure markdown renderer for delivery reports from evidence dashboard + changed file summaries.
- Add script/CLI-style command for local report generation.

### V10 Web proposal state actions
- Add pure state-action planner for safe next transitions only.
- Expose action metadata in proposal detail-compatible data.

### V11 Failed gate drilldown
- Extend evidence rows with gate-level detail for failed/passed gates.
- Surface readable failure hints for check/test/coverage/verify/build.

### V12 README verification side-effect governance
- Add helper that classifies generated demo JSON side effects so release gates can report them instead of treating them as mysterious dirty files.

### V13 Proposal/code association view
- Add pure association model that groups changed files by source/test/docs/generated and links them to proposal IDs.

### V14 Commit-ready local release check
- Add `release:local` script that runs check/test/coverage/verify/build and emits a local summary.
- No commit/push unless boss explicitly asks.
