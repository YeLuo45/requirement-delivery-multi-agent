# PRD — P-20260623-019: V22-V24 Release Local JSON Ledger Hardening

## 1. Background
V15-V21 added `release:local --json`, but the JSON payload still defaults to `proposalId: unknown`, has no CLI metadata flags, and does not persist a history record. This limits automation and audit reuse.

## 2. Features

### V22 Parameterized JSON payload
- Add `--proposal <id>` and `--title <text>` support for `scripts/release-local.mjs --json`.
- Payload should carry stable `proposalId`, `title`, generated timestamp, gates, dirty summary, ownership, and history path.

### V23 Optional history ledger write
- Add `--write-history` to persist the JSON payload under `artifacts/release-local/<timestamp>.json`.
- Create parent directories automatically.
- Keep default `--json` side-effect-free.

### V24 README documentation
- Document JSON and ledger usage in both README files.
- Keep `verify:readme` safe by treating `release:local` snippets as stub-only.
