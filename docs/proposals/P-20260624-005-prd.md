# PRD — P-20260624-005: V58-V63 Unattended Delivery Closure Completion

## 1. Background
Boss requested unattended completion of all proposed delivery-closure directions. The current release-ops stack has CLI PR draft, stable automation JSON, CI summary, and status suggestions. Remaining gaps are Web discoverability, stable Web API parity, dry-run MCP/status plans, artifact write modes, README verifier side-effect cleanup, and stronger PR draft sections.

## 2. Features

### V58 Web Release Ops closure panel
- Add a top-level Web route and nav item for Release Ops.
- Show schemaVersion, status suggestions, stage commands, PR draft, and CI summary.
- Keep it copy-friendly and visible from the header.

### V59 Web API automation parity
- Add `GET /api/release-ops/automation` returning the same stable schema as CLI `--json`.
- Keep existing `/api/release-ops` backward-compatible.

### V60 Safe status dry-run plan
- Add CLI helper output for forward-only MCP/status plans based on `statusSuggestions`.
- Default dry-run only; no proposal mutation.

### V61 README verifier side-effect cleanup
- Ensure generated demo JSON files get trailing newlines during README verification paths.
- Avoid repeated post-verify formatting noise.

### V62 CI artifact write modes
- Add `--write-json <path>` and `--write-summary <path>` for automation artifacts.
- Create parent directories and write newline-terminated files.

### V63 PR draft template enhancement
- Include Summary, Acceptance, Tests, Risk, Rollback, and Status Plan sections.
