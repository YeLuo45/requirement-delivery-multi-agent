# P-20260621-009 PRD: RDMA A+B+C+D Delivery Control Plane

## Background

RDMA already has pipeline, persistence, realtime, observability, CLI/TUI, MCP, and web surfaces. The next gap is a delivery control plane that makes real autonomous implementation safe and inspectable.

## Scope

This iteration implements A+B+C+D as one additive slice:

- A: Delivery Sandbox planning for isolated workspaces and TDD checkpoints.
- B: Collaboration access decisions for readonly/review/edit share modes.
- C: Tool governance policy decisions before file/shell/network execution.
- D: Cost/model routing with per-proposal budget accounting and downgrade behavior.

## Acceptance Criteria

- Incremental code has automated tests first and reaches >=95% coverage.
- Full `npm test` pass rate is 100%.
- `npm run coverage` passes at the configured 95% source gate.
- `npm run verify:readme` passes without README command drift.
- README documents the new package and keeps commands executable.
- No commit is created without explicit boss confirmation.

## Out of Scope

- Actually executing untrusted code in containers.
- OAuth login.
- Real provider billing API integration.
- External GitHub/Linear issue sync.
